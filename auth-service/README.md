🛡️ Auth Microservice - E-Commerce ProjectThis is the central Identity Provider (IdP) for the microservice e-commerce ecosystem. It handles user lifecycle management, secure authentication, and service-to-service authorization using asymmetric encryption.🚀 FeaturesUser Lifecycle: Secure Registration, Login, and Logout.Asymmetric Auth (RS256): Uses RSA Private/Public key pairs.Private Key: Used by this service to sign tokens.Public Key: Shared with other services to verify tokens locally.Security Hardening:Rate Limiting: Redis-backed throttling to prevent brute-force attacks.Token Blacklisting: Instant logout invalidation stored in Redis.Security Headers: HSTS, CSP, and XSS protection enabled.Observability: * Health Checks: /health/ endpoint monitoring DB and Redis connectivity.Audit Logging: Structured logging for all sensitive authentication events.Infrastructure: * Dockerized: Optimized python:3.12-slim image.CI-Ready: Standardized Makefile for one-command builds and tests.🛠️ Tech StackComponentTechnologyFrameworkDjango 5.x & Django Rest Framework (DRF)Auth StandardJWT (SimpleJWT) with RS256Primary DBPostgreSQL 15Cache/StoreRedis 7 (Blacklist & Throttling)TestingPytest (Isolated Environment)ContainerDocker & Docker Compose🚦 Getting StartedPrerequisitesDocker & Docker ComposeMake (optional, but recommended)Installation & SetupClone the repository and enter the service directory:Bashcd auth-service
Generate RSA Keys (RS256):You need a Private and Public key pair for token signing.Bashopenssl genrsa -out private.pem 2048
openssl rsa -in private.pem -outform PEM -pubout -out public.pem
Build and Start the Environment:Bashmake build
make up
Run Migrations:Bashmake migrate
🧪 TestingThe project uses an isolated testing environment to ensure development data remains clean.Bash# Run all tests (Unit, Integration, Healthchecks)
make test
